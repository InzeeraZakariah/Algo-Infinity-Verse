When to use mid±1

Apply an array and press Play or Step to begin.

Leftmost (first)

If arr[mid] >= target → hi = mid - 1 (keep searching left). Else lo = mid + 1.

Rightmost (last)

If arr[mid] <= target → lo = mid + 1 (keep searching right). Else hi = mid - 1.

Insert position

Lower bound: first index where arr[i] >= target. Same bias as leftmost; answer is final lo.

Boundary Binary Search Cheatsheet

01

Why mid±1 matters

Inclusive [lo, hi] loops need careful updates. Wrong bias causes infinite loops or missing the first/last duplicate.

02

Duplicates

Classic binary search may land on any match. Leftmost / rightmost keep going past the match to pin the boundary.

03

Insert position

Even if the target is missing, lower-bound search returns where it would be inserted to keep the array sorted.
