Recursive Centroid Engine

Halve the tree, over and over, in O(log n) levels.

Centroid decomposition repeatedly finds a subtree's centroid — the node whose removal splits it into pieces no larger than half — removes it, and recurses on what's left. The result is a separate centroid tree of depth O(log n), the standard tool behind "count all paths in a tree with property X" problems.

Generate a tree, then run the decomposition to watch centroids get found and removed level by level.

The classic centroid-decomposition worked example. For each centroid, we gather distances from the centroid to every node in its current piece, count pairs summing to ≤ K, then subtract pairs that fall entirely within the same child branch — those don't actually pass through the centroid and would otherwise be double-counted.
